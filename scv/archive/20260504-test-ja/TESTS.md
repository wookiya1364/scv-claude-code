# テストプラン — 決済ページに払い戻しボタンを追加

## 概要

払い戻しフローの E2E + 単体テスト。

## テストシナリオ

### T1. 7 日以内の払い戻し成功

- **準備**: 3 日前に決済された注文、Stripe 決済完了
- **実行**: 払い戻しボタンをクリック
- **期待**: 払い戻し処理、メール送信
- **合格条件**: HTTP 200、refund イベント発生

### T2. 7 日超過の払い戻し拒否

- **準備**: 10 日前に決済された注文
- **実行**: 払い戻しボタンをクリック
- **期待**: HTTP 403、エラーメッセージ表示
- **合格条件**: 払い戻し処理されない

## 実行方法

```bash
npm run test:e2e -- refund
```

## 合格基準

- 全シナリオ通過
- カバレッジ ≥ 80%
